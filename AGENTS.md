# AGENTS.md

## 作業ルール

- プロジェクトで作業する場合は、最初にプロジェクト内のすべての Markdown ファイルを確認する。
- 個人情報、会員名簿、学籍番号、メールアドレス、LivePocketシリアルコード、Google資産IDは記憶しない。
- 実IDや実データはコードに直書きしない。`.env`、`.clasp.json`、または管理者だけが扱うGoogle資産側で管理する。
- GitHubに共有するのは、ソースコード、テンプレート、ドキュメント、サンプルだけにする。
- 作業中に未追跡または変更済みファイルを見つけた場合、他者の変更として扱い、明示的な依頼なしに戻さない。

## 実装方針

- GASはTypeScriptで実装し、claspでApps Scriptへ反映する。
- Googleフォーム、回答スプレッドシート、会員名簿、LivePocketシリアルコードの実データは分離する。
- 抽選は半自動レビュー式とする。GASは仮当選を作成し、管理者が確認・調整した後に確定処理とメール送信を行う。
- 例外データは消さず、原因を追跡できる形で別領域に出力する。
- メール送信処理は送信上限を考慮し、送信対象数と送信済み状態を確認できる設計にする。

## セキュリティ

- `.clasp.json`、`.env`、会員名簿、フォーム回答、シリアルコード実ファイルはコミット禁止。
- ログやエラー出力に個人情報を丸ごと出さない。
- テストやサンプルには実在しないデータだけを使う。
